# Day 1 - Introduction to Salesforce

## Objective
The objective of Day 1 was to learn the fundamentals of Salesforce, understand the importance of CRM, explore Salesforce objects and records, and identify the roles of Salesforce Administrator and Salesforce Developer.

---

## Concepts Learned

### 1. Introduction to Salesforce
Salesforce is a cloud-based CRM platform used by organizations to manage customers, automate business operations, and improve communication between teams.

### 2. Understanding CRM
CRM (Customer Relationship Management) is a technology that helps businesses store customer information, track interactions, manage sales processes, and provide efficient support services.

### 3. Importance of Salesforce in Companies
Companies use Salesforce to:
- Store customer information securely
- Track sales and customer activities
- Improve communication with clients
- Automate workflows and reports
- Increase productivity and customer satisfaction

### 4. Salesforce Objects and Records
- **Object:** A structure used to store a specific type of data.
- **Record:** A single entry inside an object.

#### Example
A **Student Object** can contain:
- Student Name
- Roll Number
- Email
- Branch
- Phone Number

Each student's information is stored as a separate record.

## Submission Questions

### 1. What issue does Salesforce help solve?
Salesforce helps businesses organize customer information in a single platform. It reduces manual work, improves communication, tracks customer interactions, and provides better customer service.

### 2. Define CRM.
CRM stands for Customer Relationship Management. It is a system that helps organizations manage customer data, sales, leads, support services, and business relationships efficiently.

### 3. What is an Object in Salesforce? Give an example.
An object in Salesforce is similar to a database table used for storing information.

#### Example:
An **Employee Object** can store:
- Employee Name
- Employee ID
- Department
- Email
- Contact Number

### 4. Difference Between Salesforce Admin and Salesforce Developer

#### Salesforce Admin
- Works mainly with configuration
- Manages users and permissions
- Creates reports, dashboards, and workflows
- Uses no-code or low-code tools

#### Salesforce Developer
- Builds custom applications using code
- Works with Apex, SOQL, APIs, and Lightning Web Components
- Develops advanced automation and integrations
- Handles complex customizations

### 5. Real-World Application of Salesforce
A **Hospital Management System** can be developed using Salesforce to manage:
- Patient details
- Doctor appointments
- Medical records
- Billing information
- Support requests
